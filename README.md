### 简介
该软件用于将你的手机和电脑通过局域网连接 实现跨屏协作 并提供了实用的功能

具体更多内容可查看[Windows端](https://github.com/NativeStar/SuishoConnector-Windows)页面

### 特点
- 支持通过扫码连接和手动输入地址连接
- 在绑定设备后 只需打开软件即可自动执行连接
- 可设置在软件启动时的进行生物识别验证
- 所有传输均进行加密
- 可设置在连接中断时自动终止进程 避免不必要的内存占用和耗电
- 可选择是否信任连接的计算机 避免部分场景下泄露隐私

### 使用方法
安装软件后首次启动 右上角会有提示进行权限授权 逐条点击处理即可 除读取应用列表外的权限都可以快捷发起授权
![](/mdAssets/state_permission.png)
后续即可配合Windows端使用 通过扫描Windows端显示的二维码或输入对应地址与配对码连接
### 核心功能
- 数据互传:在双端直接互相发送文件或文本消息
- 通知转发:将手机上的通知实时转发到计算机上
- 文件浏览:可以在Windows端查看手机上的文件 并支持对部分音视频和图片格式远程查看、播放
- 信任模式:影响连接后计算机可获取的数据类型 连接新计算机后默认不信任 详见[FAQS](/FAQS.md)
- 音频转发:将设备上播放的音频传输到电脑上同步播放

### 杂项功能
- 在系统文本选择菜单中注册了'发送至计算机'快捷选项 在部分场景下提供便利 (效果如下图) 如果不需要该功能 可在设置中通过'文本快捷发送'选项关闭
![](/mdAssets/text_shortcut.png)
- 支持通过'分享'和'打开'两种文件请求进行文件发送
- 开启'自动退出'时 如果连接断开且应用处于后台 会自动终止进程
- 可设置为不保存互传记录 当应用进程彻底退出后 本次运行产生的所有互传记录将消失
- 允许设置互传接收文件保存至私有目录(/Android/data/下)或公有目录(/Download/下)
- 可设置在启动时需要进行生物识别验证 避免可能的隐私泄露(如开启该选项 自动连接会改为验证通过后才启动)
- 授予通知管理权限后 可在PC端控制Android端媒体的播放、暂停等

### 那些可能被忽略的功能 
- 互传页面长按文本消息可进行'全部复制'或'自由选择'('自由选择'需要在触发后再次长按文本消息) 长按文件消息可进行分享
- 处于互传页面时 可再次点击下方'互传'按钮回到列表底部
### 开发
项目使用纯Java实现 将源码clone到本地后通过AndroidStudio打开 等待安装依赖即可

编译方面 直接在AndroidStudio选择生成签名的apk即可
